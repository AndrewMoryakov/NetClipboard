## Безопасность сети

Актуально, если приложение используется **не на доверенной LAN/VPN** (офис, публичный WiFi, гостиничный интернет).

- [ ] **TLS поверх TCP** — сейчас сообщения идут в открытом виде. Любой в L2-сегменте через `arp-spoof` или `tcpdump` может прочитать незашифрованные элементы.
- [ ] **Аутентификация по pre-shared key (PSK)** — сейчас anyone-in-subnet:
  - может выдать себя за существующего пира (`senderId`/`senderName` подставляются произвольно в TCP-пакет),
  - может запустить свой инстанс, объявиться через UDP, и пользователь случайно отправит ему буфер через «Share to all».
  - План минимума: HMAC-SHA256 от payload с PSK; receiver проверяет MAC перед `MessageReceived?.Invoke`.
- [ ] **Опционально скрыть hostname в UDP** — сейчас имя машины броадкастится каждые 2 сек открытым текстом. Можно слать только id, а имя получать в TCP-handshake под TLS.

## Персистентность

Актуально, если буфер используется как **накапливаемая копилка** (а не разовые трансферы).

- [ ] **Сохранение истории в `%APPDATA%\NetClipboard\history.json`** — `_localItems` и `peer.Items` теряются при закрытии. Сохранять при изменении (с debounce ~2 сек) и грузить на старте.
- [ ] **Шифрование персистентного хранилища** — если master pwd задан, шифровать файл им же. Иначе — DPAPI (`ProtectedData.Protect`).
- [ ] **Опция «Не сохранять зашифрованные элементы»** — для тех, кто шифрует именно ради эфемерности.

## UX-полировка

Не критично, но заметно.

- [ ] **Status bar затирается во время Share** — `UpdateStatus()` из `CaptureClipboard`/`OnlineCheck_Tick` перезатирает «Sending to N peer(s)…». Завести флаг «есть активная операция» и не перетирать.
- [ ] **Поведение при смене master password** — старые элементы, зашифрованные предыдущим паролем, тихо становятся нерасшифровываемыми. Минимум — диалог «У вас есть N элементов под старым паролем, переchifrate?».
- [ ] **Smooth shutdown** — `OnClosing` не дожидается pending `Task.Run` (Encrypt/Decrypt). На текущем коде последствий нет, но при дальнейшем развитии может вылезти `TaskCanceledException`. Передавать `_cts.Token` в `Task.Run` и await-ить в `OnClosing`.
- [ ] **Per-item passwords UX** — нет визуальной подсказки, каким паролем зашифрован элемент (master vs custom). При расшифровке мастер пробуется автоматически, но если зашифровано кастомным — пользователь видит только «Wrong password» без подсказки попробовать другой.

## Производительность / ограничения

Низкий приоритет, edge cases.

- [ ] **Лимит на размер `ClipboardEntry.Text`** — сейчас лимит 10 МБ только на сообщение по сети (`HandleIncoming`). Локально можно скопировать 50 МБ текста, и он сядет в `_localItems`. 200 элементов × 50 МБ = 10 ГБ. Реалистично крайне редко, но обрезать до, скажем, 1 МБ с пометкой «(truncated)» — недорого.
- [ ] **PBKDF2 100k → 600k итераций** — соответствует OWASP 2023, на 2026 уже минимум. При AES-GCM с длинным паролем разница не критична. Альтернатива — переезд на Argon2id (NuGet `Konscious.Security.Cryptography.Argon2`).
- [ ] **PBKDF2 при auto-decrypt входящих** — сейчас при потоке зашифрованных сообщений каждое декриптится отдельно (по 50-100мс). Можно кэшировать derived key для master password (key = PBKDF2(master, salt) — salt разный у каждого сообщения, кэшировать нельзя по дизайну). Альтернатива — единый ключ-сессии, договариваемый при handshake (требует #PSK/TLS выше).
